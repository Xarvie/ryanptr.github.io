将QQ下线

Veröffentlicht am 2016-08-30

QQ总是不下线,给别人造成了很多误会,决定今天处理它.
先把所有移动设备账号退出后,还是显示手机在线,即使设置为隐身,qq依然显示在线,我有个疑问隐身功能设计了干什么用的?虽然我不使用它.
后来得知QQ下线得要腾讯110和微信配合使用,但我觉得这中设计并不够好,如果我是腾讯的产品设计,我会将它改为QQ+微信+360+百度+搜狗+大天使之剑同时确认才准下线才好,顺便可以加个微积分的确认验证码.这样也许会给同行带来更多的利益,挣更多的钱.
接下来到了微信下线环节,首先下载微信,微信提示需要扫码登录,再去下载一个微信PAD版,微信PAD版登陆,提示账号已被封,不遵守协议,规定时间内没有上线,故封号惩罚.
以后应该在在使用协议里面加上一条,定时不上线的用户罚款10万元整.
解封微信的条件很简单,要求是好友关系且好友绑定了银行卡,且绑定了手机号,且必须填写手机号,即可提交申诉.
条件虽然简单,但是我并不怎么使用微信,里面有没有好友还是个问题,不过还好,只打了几个电话就匹配到了一个好友.完成了验证.嗯,真不错.
上了微信账号,账号提示已解封,只需要进行好友验证我就可以上微信号了!恩,好友验证.
微信号验证成功了,进QQ离线助手,让QQ下了线,登陆110qq强制下线所有移动端,那熟悉的灰色头上终于出现了.

腾讯去年标语说说我们变了,要不忘初心,改变自己?是的,千万别忘了,当初是为了赚钱,现在一定要赚更多的钱.
End.

运行Keychain Access

Launchpad 中搜索 Keychain Access 并打开。

新建证书

  1. 菜单栏 => Certificate Assistant => Create a Certificate …
  2. Name:gdbcert
  3. Id type:self signed root
  4. Cert Type:Code Signing
  5. 勾选 Let me override…
  6. Continue
  7. SN :1
  8. VP:3650
  9. 连续7下Continue
  10. Keychain:System
  11. Create
  12. 输入密码
  13. Done

信任证书

  1. Keychain Access左列表Keychains中找到System并点击
  2. 找到gdbcert(证书)后双击弹出证书介绍框
  3. 打开Trust下拉列表，看到When using this certificate，改为Always Trust
  4. 退出并输入密码

签证

  1. 打开Terminal执行：codesign -s gdbcert /usr/local/bin/gdb
  2. 打开Launchpad搜索Activity Monitor，如果存在taskgated进程则关闭它。

新建项目HelloWorld

  4. 新建空目录 => HelloWorld\.vscode
  5. 新建空文件 => HelloWorld\.vscode\launch.json
  6. 新建空文件 => HelloWorld\.vscode\tasks.json
